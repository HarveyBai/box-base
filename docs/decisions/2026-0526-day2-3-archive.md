## 2. 关键问答

| 问题 | 关键回答 |
|---|---|
| uv.lock 为何要提交？ | 锁定依赖版本，确保所有环境依赖完全一致，是 uv 最佳实践 |
| pre-commit 路径配置为何出错？ | --directory backend 已将 CWD 切换到 backend/，entry 中不能再写 backend/，应改为相对路径 . 和 boxbase/ |
| pre-commit 提交自身配置时为何失败？ | uv add 更新了 uv.lock 但未暂存，需补充 git add uv.lock 后重新提交 |
| GitHub Actions 能拦截 --no-verify 绕过吗？ | 能。CI 在云端独立运行，不受本地 --no-verify 影响，20 秒内独立发现 mypy 错误并熔断 |
| SSH key 有密码每次都要手动输入怎么办？ | 启用 Windows OpenSSH Agent 服务，ssh-add 一次后系统级缓存，所有子进程含 AI 工具自动复用 |
| Pydantic Literal 类型有何价值？ | mypy 能通过 Literal["ok"] 约束在静态检查阶段发现业务逻辑错误，不需要等到运行时 |

.pre-commit-config.yaml 最终内容（路径修复后）：
repos:
  - repo: local
    hooks:
      - id: ruff-check
        name: ruff check
        entry: uv run --directory backend ruff check .
        language: system
        pass_filenames: false
      - id: ruff-format-check
        name: ruff format check
        entry: uv run --directory backend ruff format --check .
        language: system
        pass_filenames: false
      - id: mypy
        name: mypy
        entry: uv run --directory backend mypy boxbase/
        language: system
        pass_filenames: false

ci.yml 核心结构：working-directory: backend；steps: checkout → setup-uv@v5 → uv python install 3.12 → uv sync --all-groups → ruff check → ruff format --check → mypy boxbase/ → pytest -v --tb=short

## 5. 决策与结论

1. pre-commit hooks 使用 local repo 模式，通过 uv run --directory backend 调用项目自身工具链，不引入额外全局依赖
2. CI 使用 astral-sh/setup-uv@v5 + uv sync --all-groups，与本地环境完全一致
3. 破坏性测试确认双重防线有效：pre-commit（本地）+ GitHub Actions（云端）形成纵深防御
4. SSH passphrase 问题解决方案：Windows OpenSSH Agent 服务 + ssh-add 一次性缓存
5. TruffleHog 安全扫描未实施（Day 3 时间原因），可在 Week 2 补充

## 6. 错误与修正

| 错误 | 发现时机 | 修正方式 |
|---|---|---|
| .pre-commit-config.yaml 路径错误：ruff check backend/ 在 --directory backend 下变成 backend/backend/ | Step D 手动验证时报错 | 改为相对路径：ruff check . / ruff format --check . / mypy boxbase/ |
| pre-commit 提交自身时被拦截：uv.lock 被 uv add 更新但未暂存 | Step E 首次 commit 失败 | 删除临时文件 + 补充 git add backend/uv.lock 后重新提交 |
| AI 生成临时文件 backend/precommit_output.txt 未清理 | pre-commit 检测到未暂存文件 | 删除临时文件后重新提交 |
| SSH push 每次失败 | 每次 git push 后 | 短期：用户手动 push；长期：启用 Windows OpenSSH Agent + ssh-add |
